The **Data Link Layer (DLL)** is the **2nd layer of the OSI Model**. Its main job is to **transfer data reliably between two directly connected devices** (devices on the same network).

Think of it like a courier service:

* **Network Layer** decides **where** the package should go.
* **Data Link Layer** packs the package properly, writes the sender and receiver addresses, checks for damage, and sends it over the physical connection.

---

# Position in the OSI Model

```text
7. Application
6. Presentation
5. Session
4. Transport
3. Network
2. Data Link Layer (DLL)  ← We are here
1. Physical Layer
```

---

# What is the main job of the Data Link Layer?

The Data Link Layer ensures that data is transferred **correctly** between two devices connected by the same physical network.

Its responsibilities include:

* Framing
* Physical (MAC) addressing
* Error detection
* Flow control
* Media access control

---

# Why do we need the Data Link Layer?

Suppose Computer A wants to send data to Computer B.

The Network Layer creates an **IP packet**.

Example:

```text
Network Layer

IP Packet
```

The Physical Layer **cannot send an IP packet directly**.

The Data Link Layer first converts it into a **Frame**.

```text
Network Layer

IP Packet

↓

Data Link Layer

Frame

↓

Physical Layer

Electrical Signals
```

So the DLL acts as a bridge between the **Network Layer** and the **Physical Layer**.

---

# DLL is divided into two sublayers

The Data Link Layer has two parts:

```text
Data Link Layer

│
├── MAC (Media Access Control)
│
└── LLC (Logical Link Control)
```

Let's understand both.

---

# 1. MAC (Media Access Control)

The **MAC layer** is responsible for:

* Framing
* MAC addressing
* Controlling access to the communication medium

The book mainly discusses **Framing**, so let's start there.

---

# What is Framing?

The Network Layer gives the DLL an **IP Packet**.

Example:

```text
IP Packet

----------------------

DATA

----------------------
```

The DLL cannot send it directly.

It **wraps** the packet by adding:

* Header
* Trailer

Now it becomes:

```text
+-------------------------------+
| Header | IP Packet | Trailer  |
+-------------------------------+
```

This complete unit is called a **Frame**.

---

# Why do we add a Header?

The header contains important information.

For example:

* Source MAC Address
* Destination MAC Address
* Protocol information

Example:

```text
Header

Source MAC

Destination MAC
```

---

# What is a MAC Address?

Every network device has a **unique physical address** called a **MAC Address**.

Example:

```text
Laptop

MAC Address

00:1A:2B:3C:4D:5E
```

Another device:

```text
Printer

MAC Address

AC:45:89:FE:21:90
```

No two devices should have the same MAC address.

---

# Source MAC Address

This tells the receiver:

> "Who sent this frame?"

Example:

```text
Source MAC

00:1A:2B:3C:4D:5E
```

---

# Destination MAC Address

This tells:

> "Who should receive this frame?"

Example:

```text
Destination MAC

AC:45:89:FE:21:90
```

---

# Complete Frame

```text
+------------------------------------------------------+
| Header |              IP Packet        |   Trailer   |
+------------------------------------------------------+

Header

Source MAC
Destination MAC

Trailer

Error Checking Bits
```

---

# Why is it called Encapsulation?

Encapsulation means:

> **Wrapping one piece of data inside another structure by adding extra information.**

Example:

Imagine sending a letter.

Inside:

```text
Letter
```

You put it into an envelope.

Outside you write:

* Sender Address
* Receiver Address

Now it becomes ready for delivery.

Similarly,

```text
IP Packet

↓

Header Added

↓

Trailer Added

↓

Frame
```

This wrapping process is called **Encapsulation**.

---

# Trailer

The trailer is added at the **end** of the frame.

Example:

```text
+-----------------------------+
| Header | Data | Trailer     |
+-----------------------------+
```

The trailer mainly contains:

* Error Detection Information

Usually this is a **CRC (Cyclic Redundancy Check)** value.

---

# Why do we need Error Detection?

Sometimes data gets damaged during transmission.

For example,

Original:

```text
10110011
```

Received:

```text
10100011
```

One bit changed.

Without checking, the receiver would think the data is correct.

The trailer helps detect this problem.

---

# How does Error Detection work?

Suppose the sender calculates an error-checking value.

```text
Data

10110101

↓

CRC

1110
```

The trailer stores:

```text
Trailer

1110
```

The receiver receives:

```text
10110101

Trailer

1110
```

It calculates the CRC again.

If:

```text
Calculated CRC == Received CRC
```

✔ No error detected.

If:

```text
Calculated CRC != Received CRC
```

❌ Error detected.

---

# 2. LLC (Logical Link Control)

The **LLC layer** is responsible for:

* Flow Control
* Error Control

---

# What is Flow Control?

Imagine two people talking.

Person A speaks very slowly.

```text
A

Hello

How are you?
```

Person B understands easily.

Now imagine A speaks extremely fast.

```text
HelloHowAreYouWhereAreYouGoingBye
```

B cannot understand because the information comes too quickly.

The same thing happens in networks.

---

# Example

Suppose:

Sender speed:

```text
1000 Mbps
```

Receiver speed:

```text
100 Mbps
```

If the sender keeps sending at full speed,

```text
1000 Mbps

↓

Receiver

100 Mbps
```

The receiver's memory (buffer) becomes full.

Some data is lost.

---

# Flow Control solves this problem

The receiver tells the sender:

```text
Please slow down.
```

The sender waits before sending more data.

Example:

```text
Sender

Packet 1

(wait)

Packet 2

(wait)

Packet 3
```

This prevents buffer overflow and data loss.

---

# Everyday Example

Imagine pouring water into a bottle.

Small bottle:

```text
____
|  |
|  |
|__|
```

Big bucket:

```text
|||||||||
```

If you pour too fast,

Water spills.

Flow control is like pouring at the right speed.

---

# Error Control

Flow control manages **speed**.

Error control manages **correctness**.

It ensures that received data is accurate.

---

# Example

Original Data

```text
11110000
```

During transmission,

```text
11110010
```

One bit changes.

The trailer contains error-checking bits.

The receiver compares them.

If they don't match,

```text
Error Found
```

The receiver may request the sender to retransmit the frame (depending on the protocol in use).

---

# Difference Between Error Detection and Error Control

**Error Detection**

* Finds whether an error occurred.
* Uses techniques like CRC.

**Error Control**

* Detects the error and takes action, such as requesting retransmission or discarding the damaged frame.

---

# Difference between IP Address and MAC Address

| IP Address                   | MAC Address                  |
| ---------------------------- | ---------------------------- |
| Logical address              | Physical address             |
| Assigned by software/network | Assigned by the manufacturer |
| Can change                   | Usually remains the same     |
| Used by the Network Layer    | Used by the Data Link Layer  |
| Example: `192.168.1.10`      | Example: `00:1A:2B:3C:4D:5E` |

---

# Complete Flow of Data

```text
Application Layer
        │
        ▼
Network Layer
Creates an IP Packet
        │
        ▼
Data Link Layer
Adds Header + Trailer
Creates a Frame
        │
        ▼
Physical Layer
Converts the Frame into Signals
        │
        ▼
Transmission Medium (Cable/Fiber/Wi-Fi)
        │
        ▼
Receiver's Physical Layer
Converts Signals back into a Frame
        │
        ▼
Receiver's Data Link Layer
Checks the Trailer for Errors
Removes the Header and Trailer
Extracts the IP Packet
        │
        ▼
Receiver's Network Layer
```

## Quick Summary

| Term                           | Meaning                                                                                |
| ------------------------------ | -------------------------------------------------------------------------------------- |
| **Data Link Layer (DLL)**      | Ensures reliable communication between directly connected devices.                     |
| **MAC (Media Access Control)** | Performs framing, adds MAC addresses, and controls access to the communication medium. |
| **LLC (Logical Link Control)** | Handles flow control and error control.                                                |
| **Frame**                      | The unit of data at the Data Link Layer (Header + IP Packet + Trailer).                |
| **Header**                     | Contains information such as the source and destination MAC addresses.                 |
| **Trailer**                    | Contains error-detection information (such as CRC).                                    |
| **Encapsulation**              | The process of adding a header and trailer around an IP packet to create a frame.      |
| **MAC Address**                | A unique physical address used to identify devices on the same network.                |
| **Flow Control**               | Prevents the sender from transmitting data faster than the receiver can handle.        |
| **Error Detection**            | Detects whether data has been corrupted during transmission.                           |
| **Error Control**              | Handles detected errors, often by requesting retransmission of damaged frames.         |

**Exam definition:**

> **The Data Link Layer (Layer 2 of the OSI model) provides reliable data transfer between directly connected devices. It converts IP packets into frames by adding a header and trailer, uses MAC addresses for local delivery, performs error detection, and provides flow and error control through its two sublayers: MAC and LLC.**
