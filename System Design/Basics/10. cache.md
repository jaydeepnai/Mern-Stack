# Cache Memory — Notes

## Definition
Cache memory is a **small, high-speed memory** located close to the CPU, used to store copies of recently/frequently accessed instructions and data. Its purpose is to **reduce access time** and improve overall system performance.

---

## Core Principle: Locality of Reference

Cache works because programs don't access memory randomly — they tend to reuse a small subset of data repeatedly. This is called **Locality of Reference**, and it has two types:

| Type | Meaning |
|---|---|
| **Temporal Locality** | If a memory location is accessed once, it's likely to be accessed **again soon** (same data reused repeatedly). |
| **Spatial Locality** | If a memory location is accessed, **nearby locations** are likely to be accessed soon after (data near each other tends to be used together). |

---

## Cache Hit vs Cache Miss

When the CPU needs data, it checks the cache first:

- **Cache Hit** → Data found in cache → fast retrieval.
- **Cache Miss** → Data not found in cache → must fetch from a lower memory level (e.g., main memory/RAM), and this data is then placed into the cache for future use.

---

## Cache Organization & Memory Hierarchy

- When a cache miss occurs, an entire **block** of data (not just a single item) is transferred from main memory to cache — based on spatial locality, since nearby data is likely to be needed next.
- Goal: keep the **most frequently accessed data** in cache so the CPU rarely has to go to slower main memory.

### Cache Lines
- Cache is organized into **cache lines**, not individual bytes.
- Each cache line typically holds **16–64 bytes** of contiguous data.
- Cache lines don't have fixed addresses — they can hold data from different memory locations at different times.

---

## Methods for Filling Cache Lines

| Method | Description |
|---|---|
| **Fully Associative** | Most flexible — any memory block can go into **any** cache line. |
| **Direct Mapped** | Simplest — each memory block maps to **exactly one specific** cache line. |
| **Set Associative** *(referred to as "Self Associated")* | A middle ground — combines both approaches, offering some flexibility with mapping constraints. |

---

## Measuring Cache Performance — Hit Ratio

```
Hit Ratio = Number of Hits / Total Accesses = hits / (hits + misses)
```

A **higher hit ratio** = better performance (CPU retrieves data from fast cache more often instead of slow main memory).

### Ways to Improve Cache Performance
- Increase **cache block size**
- Increase **cache associativity**
- Reduce the **miss rate** (fewer cache misses)
- Reduce the **miss penalty** (time taken to fetch data from main memory after a miss)
- Reduce the **hit time** (time to access data already in cache)

---

## Quick Recap
- **Cache** = small, fast memory near CPU storing frequently used data.
- Works because of **Locality of Reference** (Temporal + Spatial).
- **Hit** = found in cache (fast) | **Miss** = not found, fetched from main memory (slow).
- Data moves in **blocks/cache lines**, not single bytes.
- Cache line filling: **Fully Associative**, **Direct Mapped**, or **Set Associative**.
- Performance measured via **Hit Ratio** — improved by tuning block size, associativity, miss rate, and miss penalty.
